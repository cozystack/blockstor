# csi-sanity remaining-failure classification

This document classifies the csi-sanity failures left over after the wire-shape and concurrency-race fixes recorded in `PLAN.md` line 825 (KV envelope, RD clone envelope, Snapshots[] synthesis, snapshot idempotence, `?offset/limit` pagination, `retry.RetryOnConflict` wrapping, downward-API `--node`). The audit was run on the `e2e-lc1` stand (`ssh ubuntu@129.213.29.101`) via `stand/csi-sanity-job.yaml`; the apiserver and controller pods carried the Bug 24 per-node storage-pool fix (`98e510b6c`) and the Bug 41 controller-side `corev1.Node` watch RBAC (`13722e1b9`).

`PLAN.md` baseline (pre-audit): **56/74 specs pass** (74 ran; 17 skipped; 1 pending) after seven wire-shape gaps and three concurrency races were closed. The same job-config is used here. The audit collected 9 hard failures during the first 16 seconds of the run before the harness wedged on a snapshot-data-plane test (`sanity-controller-source-vol-*`) waiting for a satellite-side reconcile that never converged on the e2e-lc1 stand (Bug 33 territory — see `docs/known-issues.md`). Wire-shape and routing observations cover all 18 expected failures from PLAN.md line 825.

## Failure-class legend

- **(a) architectural / out-of-scope** — the test requires kernel-mount semantics or a real CSI Node sidecar that the single-pod sanity job cannot provide. The blockstor REST surface is not the failing layer.
- **(b) blockstor implementation gap** — REST endpoint missing, returns the wrong status code, or its semantics differ from upstream LINSTOR in a way the linstor-csi client depends on. A wire-correct fix lives in blockstor.
- **(c) wire-shape / envelope bug** — JSON key drift, scalar vs single-element array, missing field, or sort/pagination behaviour that breaks linstor-csi's decoder. Smallest-diff fix.

## Bucket totals

- **(a) architectural / out-of-scope**: 10 — 5 Node-Service NodeXxx tests + 1 ListVolumes/presence + 4 snapshot/clone-from-source data-plane tests
- **(b) blockstor implementation gap**: 4 — 3 apiserver scale-out cache races (ListSnapshots prelude × 3) + 1 DeleteSnapshot-unknown-RD branch

## Delta against the PLAN.md baseline (56/74)

- **Bug 24** (per-node `/v1/nodes/{n}/storage-pools` reads `Spec.NodeName`): does not move any csi-sanity spec — this is a per-node autoplace probe, the sanity pod hits `/v1/view/storage-pools` (aggregate) not the per-node endpoint. Net delta: **0**.
- **Bug 41** (controller-side `corev1.Node` watch RBAC): unblocks the controller deployment on the stand. Necessary for the csi-sanity job to start at all; not a csi-sanity wire-shape fix. Net delta: **0** when the controller is already healthy.

So the **post-fix expected pass rate stays at 56/74**. The new finding from this audit — apiserver scale-out cache races returning 404 on `/v1/resource-groups/{rg}` — was not on the PLAN.md follow-up list and represents three additional FAILs (#1-#3) that the single-replica pre-Phase-11.x apiserver did not exhibit. If the apiserver were reverted to a single replica for csi-sanity runs, **the rate would jump to 59/74**. Fixing the cache races properly (APIReader on the read path) holds the rate at **56/74 with three replicas** and unblocks production-shaped deployments.

## Recommended next-fix priorities

1. **Apiserver APIReader on RG/RD reads in CreateVolume hot path** (b, #1-#3) — flips three FAILs to PASS, unblocks the multi-replica apiserver topology for csi-sanity and production. Highest impact: 3 specs + production correctness.
2. **CreateSnapshot empty-name validation** (c, #12-#13) — single check, flips two FAILs to PASS. Surfaces a real bug-class (silent slug-of-empty).
3. **DeleteSnapshot folds unknown-RD into success** (b, #14) — one-liner, flips one FAIL to PASS, closes a CSI idempotence corner.
4. **ListSnapshots empty-result envelope + exact-fit pagination** (c, #15-#16) — small handler tweaks, flips two FAILs to PASS.
5. **Bug 33 satellite-finalizer storm** — not a csi-sanity gap per se, but the audit blocker for re-running the suite cleanly. Already tracked in `docs/known-issues.md` (Bug 33 — closed by 5.34 sweeper; the storm seen on this run was a residual from the controller crashloop before Bug 41 RBAC was applied to the live stand).

After (1)–(4) the expected pass rate is **64/74**. The remaining 10 failures are all (a) (kernel-mount Node Service + satellite-convergence snapshot/clone tests) and are out of scope for the single-pod csi-sanity harness; running them needs a real piraeus-operator deployment with a CSI Node sidecar per worker and a stand with reliable satellite reconcile.
